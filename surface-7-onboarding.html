DISMISS RULE · /referrals?tour=done

Trigger spec

FIRES WHEN
A curated commish hits /referrals for the first time after their wallet is bound. Auth state + seen_tour=false on user record.
STORAGE
User record: seen_tour, tour_completed_at, tour_step. Resumes mid-step if they close before step 04.
DISMISS RULE
"X" sets seen_tour=true but leaves a CTA in the dashboard kebab. Auto-completes only after first share + first qualifying trade attributed.
EDGE
If the commish was somehow attributed as a referee elsewhere, the welcome modal degrades to a single-step "you're attributed AND a commish — here's both surfaces" view (not designed in v0).
MOBILE
Steps stack full-bleed. Highlighted dashboard regions still render below the modal but full-screen. Swipe → → → between steps. Same dismiss rule.
